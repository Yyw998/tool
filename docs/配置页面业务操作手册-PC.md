# 入职资料收集工具配置页面操作手册（PC 版）

适用对象：HR、招聘运营、人事接口人、负责维护入职资料清单的业务同事  
适用页面：配置管理页 `config.html`、候选人资料收集页 `index.html`

---

## 一、这套工具是做什么的？

这套工具用于让候选人按要求填写基础信息、上传入职资料，并在自己的浏览器本地生成加密资料包。HR 可以通过“配置管理页”维护候选人看到的表单内容，例如：

- 页面标题和提示语；
- 需要候选人填写哪些基础信息；
- 需要候选人上传哪些资料；
- 每项资料是否必填、允许上传什么文件、最多上传几个；
- 模板文件和示例图片；
- ZIP 资料包名称；
- 邮件收件人、抄送人、邮件标题和正文。

候选人不会进入配置页面，只会访问资料收集页面。

---

## 二、最重要的生效规则

配置页面修改完成后，不是自动保存到线上页面。

正确流程是：

1. 打开配置管理页；
2. 修改配置；
3. 点击右上角“导出发布配置”；
4. 得到一个 `config.js` 文件；
5. 在静态托管平台中替换站点目录下的 `js/config.js`；
6. 候选人刷新资料收集页面后，即可看到最新配置。

简单理解：

```text
配置页面修改内容
  ↓ 导出发布配置
config.js
  ↓ 替换站点 js/config.js
候选人资料收集页生效
```

> 注意：候选人页面读取的是站点中的 `js/config.js`，不是某个浏览器里的临时配置。

## 三、开始配置前需要准备什么？

正式维护前，建议先准备好以下信息：

| 准备项 | 说明 |
| --- | --- |
| 配置管理页地址 | 通常是 `你的站点地址/config.html` |
| 候选人表单页地址 | 通常是 `你的站点地址/index.html` |
| 托管平台管理权限 | 用于替换线上 `js/config.js` |
| 模板文件 | 例如 `中粮贸易新员工信息登记表.docx`，放到站点 `templates` 目录 |
| 示例图片 | 例如 `身份证示例.png`，放到站点 `examples` 目录 |
| HR 收件邮箱 | 候选人最终发送资料包的邮箱 |
| 抄送邮箱 | 可选 |
| 邮件标题和正文要求 | 候选人最后一步会复制使用 |

## 六、表单项配置

“表单项配置”是最常用的配置区域，分为两部分：

1. 基础信息项；
2. 资料上传项。

### 6.1 基础信息项

基础信息项是候选人需要填写的文字信息，例如姓名、手机号。

页面上每一行代表一个字段。

| 配置项 | 说明 |
| --- | --- |
| 字段名称 | 候选人看到的字段名称，例如“姓名” |
| 必填 | 打开后，候选人必须填写该项才能进入后续步骤 |
| 占位提示 | 输入框里的提示文字，例如“请输入姓名” |
| 排序 | 控制字段显示顺序，数字越小越靠前 |
| 删除 | 删除该字段 |

#### 新增基础字段

操作方法：

1. 在“基础信息项”下方点击“新增字段”；
2. 填写字段名称；
3. 设置是否必填；
4. 填写占位提示；
5. 设置排序；
6. 导出发布配置。

示例：

| 字段名称 | 是否必填 | 占位提示 | 排序 |
| --- | --- | --- | --- |
| 身份证号 | 是 | 请输入身份证号 | 3 |

#### 删除基础字段

操作方法：

1. 找到要删除的字段；
2. 点击右侧“删除”；
3. 确认该字段确实不再需要；
4. 导出发布配置。

> 建议：姓名、手机号通常不要删除，因为后续 ZIP 文件名和邮件正文会用到。

| 配置项 | 说明 | 示例 |

### 6.4 资料项名称怎么填？

“资料项名称”会直接显示给候选人，也会用于资料包内文件夹/文件命名。

建议名称简短、明确，例如：

- 身份证；
- 政审报告；
- 学历证书；
- 学历学信网验证报告；
- 学位证书；
- 学位学信网验证报告；
- 体检报告电子件；
- 无犯罪记录证明；
- 上一家单位离职证明；
- 中粮贸易新员工信息登记表；
- 其他材料。

不建议写太长，例如：

```text
请上传身份证正反面照片并确保清晰完整
```

这种内容更适合写到“资料项说明”里。

---

### 6.5 显示顺序怎么用？

显示顺序决定候选人页面里的资料排列顺序。

规则：

- 数字越小越靠前；
- 建议从 1 开始顺序填写；
- 如果两个资料项顺序重复，页面仍能显示，但不利于维护。

示例：

| 资料项 | 显示顺序 |
| --- | --- |
| 身份证 | 1 |
| 政审报告 | 2 |
| 学历证书 | 3 |

---

### 6.6 是否必填怎么设置？

“是否必填”打开后，候选人必须上传该资料。

建议：

| 场景 | 设置 |
| --- | --- |
| 必须提交的入职资料 | 打开 |
| 暂时不确定是否需要 | 关闭 |
| 只针对部分人员需要 | 关闭，并在资料项说明里写清楚适用情况 |

例如“其他材料”通常可以设为非必填。

---

### 6.7 允许上传类型怎么选？

点击“允许上传类型”下拉框，可以选择候选人允许上传的文件类型。

常见选择：

| 资料类型 | 建议上传类型 |
| --- | --- |
| 身份证、银行卡、照片 | 图片、PDF |
| 证书类资料 | 图片、PDF |
| 表格或登记表 | Word 文档、Excel 表格、PDF |
| 扫描件 | 图片、PDF |

如果不确定，建议选择：

```text
图片 + PDF
```

这样候选人可以上传照片，也可以上传扫描 PDF。

---

### 6.8 最多上传文件数怎么填？

用于限制某项资料最多上传几个文件。

示例：

| 资料项 | 建议最多上传文件数 |
| --- | --- |
| 身份证 | 4 |
| 学历证书 | 6 |
| 学信网验证报告 | 6 |
| 其他材料 | 20 |

如果资料可能包含多个文件，可以适当调大。

---

### 6.9 单文件大小 MB 怎么填？

用于限制单个上传文件的大小。

一般建议保持默认：

```text
20 MB
```

如果候选人上传高清扫描件较多，可以适当增加。

---

### 6.10 模板文件名称怎么填？

如果某项资料需要候选人下载模板填写，例如“新员工信息登记表”，需要配置模板文件名称。

操作前请先把模板文件放到站点目录：

```text
templates/
```

配置时只填写文件名，不要填写完整路径。

正确示例：

```text
中粮贸易新员工信息登记表.docx
```

不建议填写：

```text
templates/中粮贸易新员工信息登记表.docx
```

配置后可以点击右侧“预览”按钮检查文件是否能打开。

如果该资料没有模板，留空即可。

---

### 6.11 示例图片名称怎么填？

如果希望候选人在准备资料前看到示例图，需要配置示例图片名称。

操作前请先把示例图片放到站点目录：

```text
examples/
```

配置时只填写文件名，不要填写完整路径。

正确示例：

```text
身份证示例.png
```

配置后可以点击右侧“预览”按钮检查图片是否能正常显示。

如果该资料没有示例图，留空即可。

---

### 6.12 资料项说明怎么写？

资料项说明会展示给候选人，用于告诉候选人该资料怎么准备。

建议写得短、明确、可执行。

推荐写法：

```text
1. 请提交身份证正反面合并在同一页的图片；
2. 报到现场需携带身份证原件。
```

不推荐写法：

```text
请按照要求上传相关材料。
```

原因：候选人不知道“相关材料”具体是什么。

---

### 6.13 资料项命名规则要不要改？

一般不建议业务人员修改，保持默认即可。

默认规则：

```text
{姓名}_{手机号}_{附件编号}_{资料名称}_{序号}.{扩展名}
```

生成后的文件名示例：

```text
张三_13800138000_附件01_身份证_01.jpg
```

如果确实要改，请确认变量名称不要写错。

可用变量：

| 变量 | 含义 |
| --- | --- |
| `{姓名}` | 候选人填写的姓名 |
| `{手机号}` | 候选人填写的手机号 |
| `{附件编号}` | 系统生成的附件编号 |
| `{资料名称}` | 资料项名称 |
| `{序号}` | 同一资料项下第几个文件 |
| `{扩展名}` | 文件后缀 |

---

### 6.14 新增一个资料上传项

例如要新增“劳动合同签署确认单”。

操作步骤：

1. 进入“表单项配置”；
2. 找到“资料上传项”区域；
3. 点击底部“新增附件项”；
4. 新增项会自动展开；
5. 填写“资料项名称”：劳动合同签署确认单；
6. 设置“显示顺序”；
7. 设置“是否必填”；
8. 选择“允许上传类型”；
9. 设置“最多上传文件数”和“单文件大小 MB”；
10. 如有模板，填写模板文件名称；
11. 如有示例图，填写示例图片名称；
12. 填写资料项说明；
13. 检查无误后，点击“导出发布配置”。

---

### 6.15 删除一个资料上传项

操作步骤：

1. 找到要删除的资料项；
2. 点击右侧“删除”；
3. 确认该资料后续不再需要；
4. 点击“导出发布配置”；
5. 替换线上 `js/config.js`。

> 删除前建议先确认该资料项是否仍在入职流程中使用，避免候选人漏交资料。

## 七、打包核查配置

进入“打包核查配置”页签，可以设置候选人生成 ZIP 资料包时的规则。

| 配置项 | 说明 | 建议 |
| --- | --- | --- |
| ZIP 文件名规则 | 候选人下载资料包时的文件名 | 一般保持默认 |
| ZIP 密码提示 | 候选人输入压缩包密码时看到的提示 | 写清楚密码由 HR 另行约定 |

默认 ZIP 文件名规则：

```text
{姓名}_{手机号}_入职资料包_{日期}.zip
```

生成示例：

```text
张三_13800138000_入职资料包_20260627.zip
```

ZIP 密码提示示例：

```text
请输入人事约定的 ZIP 解压密码。页面不会保存该密码。
```

## 八、邮件配置

进入“邮件配置”页签，可以维护候选人在最后一步看到的邮件发送信息。

| 配置项 | 说明 |
| --- | --- |
| 收件邮箱 | 候选人需要发送资料包的 HR 邮箱 |
| 抄送邮箱 | 可选，没有可留空 |
| 邮件标题模板 | 候选人邮件标题 |
| 邮件正文模板 | 候选人邮件正文 |

### 8.1 收件邮箱

填写 HR 收件邮箱，例如：

```text
hr@example.com
```

如果暂时不知道，可以留空。候选人页面会提示“请联系 HR 获取收件邮箱”。

### 8.2 抄送邮箱

如需抄送多个邮箱，请用英文逗号分隔：

```text
hr1@example.com,hr2@example.com
```

如果不需要抄送，留空即可。

### 8.3 邮件标题模板

推荐格式：

```text
【入职资料】{姓名}-{手机号}
```

生成示例：

```text
【入职资料】张三-13800138000
```

### 8.4 邮件正文模板

推荐正文：

```text
您好：

我的入职资料已整理为加密压缩包，请查收附件。

姓名：{姓名}
手机号：{手机号}
资料包名称：{压缩包名称}

压缩包已加密，解压密码请使用约定密码。

谢谢。
```

## 九、发布配置

所有配置修改完成后，需要发布给候选人。

### 9.1 导出发布配置

操作步骤：

1. 检查四个页签配置是否完成；
2. 点击页面右上角“导出发布配置”；
3. 浏览器会下载一个文件，文件名为：

```text
config.js
```

【截图 8：导出发布配置按钮】

### 9.2 替换线上配置文件

在静态托管平台中找到站点文件：

```text
js/config.js
```

用刚刚下载的 `config.js` 替换它。

替换完成后，候选人刷新资料收集页面，即可读取新配置。

> 注意：请确认替换的是 `js/config.js`，不是根目录下的其他文件。

### 9.3 同步上传模板和示例图片

如果配置里填写了模板文件名称，需要把模板文件上传到：

```text
templates/
```

如果配置里填写了示例图片名称，需要把示例图片上传到：

```text
examples/
```

示例：

| 配置中填写 | 文件应放到 |
| --- | --- |
| 中粮贸易新员工信息登记表.docx | `templates/中粮贸易新员工信息登记表.docx` |
| 身份证示例.png | `examples/身份证示例.png` |

## 十、发布后如何检查候选人页面？

发布后建议按候选人的视角完整检查一次。

检查步骤：

1. 打开候选人资料收集页 `index.html`；
2. 查看引导页标题、副标题是否正确；
3. 点击“立即开始填写”；
4. 检查第一步“准备资料”里的资料项、说明、示例图是否正确；
5. 点击下一步；
6. 检查基础信息项是否正确；
7. 检查资料上传项是否正确；
8. 上传一个测试文件；
9. 输入 ZIP 密码；
10. 生成并下载资料包；
11. 打开 ZIP 核查文件清单；
12. 进入邮件发送步骤；
13. 检查收件邮箱、抄送人、邮件标题、邮件正文是否正确。

## 十一、常见问题

### 11.1 修改配置后，候选人页面没有变化

可能原因：

| 原因 | 处理方法 |
| --- | --- |
| 没有点击“导出发布配置” | 回到配置页导出 `config.js` |
| 没有替换线上 `js/config.js` | 在托管平台替换对应文件 |
| 替换错目录 | 确认路径是 `js/config.js` |
| 浏览器缓存 | 刷新页面后再检查 |

### 11.2 模板预览打不开

检查：

1. 模板文件名称是否填写正确；
2. 文件是否已放到 `templates/` 目录；
3. 文件后缀是否正确，例如 `.docx`；
4. 文件名是否有错别字或多余空格。

### 11.3 示例图片不显示

检查：

1. 示例图片名称是否填写正确；
2. 图片是否已放到 `examples/` 目录；
3. 图片格式是否为浏览器可打开的格式，例如 `.png`、`.jpg`；
4. 文件名是否完全一致。

### 11.4 候选人无法上传某类文件

检查该资料项的“允许上传类型”。

例如候选人要上传 Word 文件，但配置里只选了“图片、PDF”，则需要增加“Word 文档”。

### 11.5 候选人提示文件数量超过限制

检查该资料项的“最多上传文件数”，适当调大。

### 11.6 候选人提示文件太大

检查该资料项的“单文件大小 MB”，适当调大。

## 十三、推荐维护原则

为了让候选人体验更好，建议遵循以下原则：

1. 资料项名称尽量短；
2. 资料项说明要具体，不要写空泛提示；
3. 必填项不要设置过多，确实必须提交再设置为必填；
4. 能给示例图的资料尽量给示例图；
5. 有模板的资料项一定要配置模板文件；
6. 每次发布后都从候选人页面完整测试一遍；
7. ZIP 解压密码不要写在邮件正文里，建议通过微信单独告知。
